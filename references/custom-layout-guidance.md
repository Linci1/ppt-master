# 自由布局（custom）类型指引

> 面向 Executor_Security 的"无模版"页面生成规范。当标准布局（img_right/left/top/bottom/split）和 L3 套路提示词均不匹配当前内容时，启用 custom 布局。

## 1. 何时启用 custom

### 1.1 触发条件

以下任一条件满足时，使用 `layout_type: "custom"`：

| 条件 | 示例 |
|------|------|
| 页面内容无法归入 5 种标准布局 | 多面板仪表盘、不规则图文混排 |
| L3 套路提示词无匹配项 | 既不是攻击链复盘，也不是风险评估摘要等既定套路 |
| 需要 3+ 独立信息区域 | 3 栏布局、2×2 四象限、上中下三段 |
| 图表与文字需自由拼合 | SVG 图表嵌入文字流中，非标准 img_right 范式 |
| 布局需精确像素级控制 | 图标阵列、时间轴动画页、信息图谱 |

### 1.2 不应使用 custom 的场景

- 标准 layout_type 可覆盖的 → 优先用标准布局
- L3 套路提示词有匹配的 → 套路模板覆盖规则优先
- 仅需微调标准布局 → 用 `layout_gravity` 和 `content_ratio` 即可

## 2. 坐标系与网格

### 2.1 画布基准

```
画布尺寸: 1280×720 px  (10"×5.625" @ 128dpi)
原点: 左上角 (0, 0)
标题区: y = 30~90 px (约 0.23"~0.7")
正文区: y = 100~680 px
页脚区: y = 680~710 px
安全边距: x 方向最小 60px，y 方向最小 40px
```

### 2.2 推荐网格

```
6列栅格 (1280px):
  col_1:  60~235   (175px)
  col_2:  255~430  (175px)
  col_3:  450~625  (175px)
  col_4:  655~830  (175px)
  col_5:  860~1035 (175px)
  col_6:  1050~1220(170px)
  列间距: 20px

4行栅格 (正文区 100~680):
  row_1: 100~230   (130px)
  row_2: 245~380   (135px)
  row_3: 395~530   (135px)
  row_4: 545~680   (135px)
  行间距: 15px
```

### 2.3 通用约束

```
最小元素尺寸: 80×60 px
最大图文比例: 图片区域 ≤ 60% 总画布面积
文字最小字号: 11px (8.6pt)
推荐正文行距: 1.4em
最大信息密度: ≤ 6 个信息块 + ≤ 4 张小截图 (98% 小截图遵循)
```

## 3. 安服定制范式

### 3.1 仪表盘页 (Dashboard)

适用：安全态势总览、SOC 运营日报。

```
┌─────────────────────────────────────┐
│           页面标题                    │
├──────────┬──────────┬───────────────┤
│  关键KPI │  风险趋势 │  告警统计      │
│  3卡片   │  折线SVG │  柱状SVG       │
│  col_1   │  col_2-3 │  col_4-6      │
├──────────┴──────────┴───────────────┤
│  最近事件列表 (col_1-6)              │
│  3行表格：时间/事件/等级/状态         │
└─────────────────────────────────────┘
```

颜色：品牌绿 #7BBD4A KPI 数值，告警红 #FF0000 高亮危险项
图片：右侧折线/柱状 SVG 图（小面积）
禁用：大图、全画布图

### 3.2 红蓝对比页 (Red-Blue)

适用：攻防演练对比、方案优劣对比。

```
┌─────────────────────────────────────┐
│           页面标题                    │
├──────────────────┬──────────────────┤
│   红队视角        │   蓝队视角        │
│   col_1-3         │   col_4-6        │
│   攻击路径         │   防御策略        │
│   关键技术         │   检测手段        │
│   时间线(小)       │   响应流程        │
└──────────────────┴──────────────────┘
```

颜色：左栏 告警红 #C00000 边框/accent，右栏 品牌绿 #7BBD4A
图片：各侧 1 张小截图（攻击链截图 / 防御架构图），右下角
禁忌：不要混合双色在一侧

### 3.3 威胁概览页 (Threat Landscape)

适用：威胁态势分析、APT 组织画像。

```
┌─────────────────────────────────────┐
│           页面标题                    │
├─────────────────────┬───────────────┤
│  威胁地图 / 气泡图    │  关键发现      │
│  col_1-4             │  col_5-6      │
│  风险气泡 SVG         │  3条要点       │
│  (居中大图)          │  带图标列表     │
├─────────────────────┴───────────────┤
│  威胁统计条: 高危3 中危5 低危12      │
│  col_1-6 (单行数字条)                │
└─────────────────────────────────────┘
```

颜色：告警色梯度（深红→红→橙→灰）
图片：气泡图 SVG 占据左侧 60%，可突破 img_right 范式
特殊许可：主图面积可达 30%（常规 10% 限制在此范式例外）

### 3.4 整改建议页 (Remediation Plan)

适用：安全整改计划、漏洞修复优先级。

```
┌─────────────────────────────────────┐
│           页面标题                    │
├────┬────────────────────────────┬───┤
│  # │  整改项                     │优先级│
│  1 │  漏洞修复详情 ...           │ 🔴  │
│  2 │  配置加固详情 ...           │ 🟠  │
│  3 │  策略优化详情 ...           │ 🟡  │
├────┴────────────────────────────┴───┤
│  整改时间线: ■■■■■■■□□□ 70%         │
└─────────────────────────────────────┘
```

颜色：优先级 🔴C00000 🟠ED7D31 🟡F0A020
图片：每条可配 1 张小截图（漏洞截图/配置截图）右侧
列表项：最多 5 条，每条 3 行

## 4. 图片策略（custom 布局特殊规则）

### 4.1 强制规则

1. **95% 页面必须有图片** — 比标准布局的 95% 规则更严：custom 页不能出现纯文字
2. **图片必须来自源文档** — 从文档提取的真实截图，不得使用占位色块
3. **优先 img_right** — 即使 custom，图片仍倾向右侧/右下角
4. **小截图优先** — 单张图片面积 ≤ 15% 总画布（除威胁概览范式例外）

### 4.2 图片嵌入语法

```yaml
# 标准方式
images:
  - src: "../images/screenshot-01.png"
    position: [1100, 120]      # [x, y] 左上角坐标
    size: [150, 110]           # [w, h]
    label: "攻击载荷截图"

  - src: "../images/vuln-list.png"
    gravity: "bottom-right"    # 语义定位
    max_width: 200             # 最大宽度约束
    margin: [20, 20]           # 距边距
```

### 4.3 图片下限

```
最少图片数: 1 张 (custom 硬性要求)
最多图片数: 4 张 (防信息过载)
推荐: 2 张 (1 主图 + 1 辅图)
```

## 5. 颜色纪律（custom 不豁免）

| 元素 | 颜色 | 说明 |
|------|------|------|
| 正文文字 | #404040 | 12pt，永不变 |
| 页面背景 | #FFFFFF | 正文页白底铁律 |
| 标题色 | #1A1C1E | 深灰，非纯黑 |
| 主强调色 | #7BBD4A | 品牌绿（统一） |
| 告警/高危 | #C00000 / #FF0000 | 仅用于风险相关高亮 |
| 中危 | #ED7D31 | 橙色 |
| 低危/安全 | #4CAF50 | 绿色 |
| 图表辅助线 | #E0E0E0 | 浅灰，不抢眼 |
| 禁用色 | 暗底背景、霓虹色、超过 3 种强调色 | — |

## 7. Executor 执行前检查清单

对每个 custom 布局页面，生成 SVG 前必须逐项确认：

```
☐ 页面背景是否为纯白 #FFFFFF？
☐ 正文文字颜色是否为 #404040 12pt？
☐ 品牌色使用是否不超过 2 种？
☐ 告警色是否仅用于风险相关元素？
☐ 图片是否来自源文档提取（非占位色块）？
☐ 是否有至少 1 张真实图片？
☐ 图片位置是否在右侧/右下角（优先）？
☐ 小截图面积是否 ≤ 15% 总画布？
☐ 信息密度是否在 6 块以内？
☐ grid 声明是否与 panels 布局一致？
☐ 是否避免了暗底背景？
☐ 讲稿文本是否中文、分段分点、风险量化？
```

## 8. 与标准布局的关系

```
标准布局 (layout_type)
  ├── img_right / img_left / img_top / img_bottom / split
  │   → 常规正文页首选
  │
  ├── L3 套路提示词 (routine_override)
  │   → 安服专用叙事模式（攻击链/风险评估/合规等8种）
  │
  └── custom ★ (本文档)
      → 以上均不匹配时使用
      → 注意: custom 仍需遵循颜色纪律 + 图片规则
      → 禁止: custom 中重新实现标准布局已有的模式
```

## 9. Pitfalls（常见错误）

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| custom 页忘记放图片 | 白板页，空洞 | 至少 1 张源文档截图 |
| 用 #F8F9FA 灰底 | 破坏白底一致性 | 背景永远是 #FFFFFF |
| 超过 6 个信息块 | 视觉过载，信息失效 | 拆分到多页 |
| 页面顶部没标题 | 导航断裂 | 必须有 y<90px 的标题 |
| 图片过大 (>30%) | 比例失衡，像图片页 | 图片 ≤ 15% (范式例外除外) |
| 同时用两种品牌色 | 风格割裂 | 统一用 #7BBD4A 品牌绿 |
| custom 页放固定页内容 | 违反固定页规则 | 固定页仅用于封面/章节/结尾 |
