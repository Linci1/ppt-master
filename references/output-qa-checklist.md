# 输出 QA 检查清单

在**最终交付阶段**使用本清单。只要有任一适用项未通过，就不要交付 PPT。

## A. 固定骨架与模板保真

- 如果所选模板带有 `qa_profile.md` 或 fixed skeleton 说明，必须保留其页眉、页脚、Logo 区、目录骨架、遮罩层和模板装饰条。
- 允许灵活变化的是正文内容区的卡片、图表、时间线、矩阵、图文组合，而不是把模板品牌区整体挪位。
- 如果模板声明了 `data-brand-required` / `data-brand-assets`，则必须同时确认：
  - Logo 没有缺失
  - Logo 使用的是批准资产
  - Logo 没有被额外加白底、描边或底板
- 对“长亭通用墨绿色”（模板 ID：`chaitin`），重点确认：
  - 内容页左下 Logo 未与正文冲突
  - 底部装饰条上方仍有呼吸空间
  - 目录页 2x2 骨架没有被拉散
  - 目录项结构保持一致（全带副标题或全不带副标题）
- 对“长亭安服”（模板 ID：`security_service`），额外确认：
  - 浅底页使用绿色版 Logo，深色/背景图页使用白色版 Logo
  - 顶部右侧 Logo 与标题、图示、标签没有打架

## B. 文本适配

- 任何单行文本都不能越过卡片、面板或页头边界。
- 卡片内部不同文本块不能互相覆盖；即使都还在外框内部，只要正文、注释、分隔线区域发生碰撞，也视为未通过。
- 密集卡片必须有足够高度容纳所有换行文本，不能指望 PowerPoint 自动缩放兜底。
- 如果页面上长 bullet 太多，应放大容器、拆分信息块或改布局，而不是硬塞进同一张卡片。
- 在后处理前必须运行：

```bash
python3 scripts/check_svg_text_fit.py <project_path>/svg_output
```

- 在 `finalize_svg.py` 之后必须再运行一次：

```bash
python3 scripts/check_svg_text_fit.py <project_path>/svg_final
```

不能只检查 `svg_output/` 就直接交付。

## C. 密集页面

以下页面属于高风险密集页，必须重点检查：

- 问题汇总页
- 机制概览页
- 要求 / 政策页
- 流程 / onboarding 页
- 多卡片证据页

重点确认：

- 卡片之间没有视觉碰撞
- 文本不会显得拥挤
- 标题与正文之间仍有足够留白
- 正文最后一行不会贴近卡片底边
- 卡片内部的分隔线、装饰条、底部边框栏不能压住任何正文
- 即使没有技术性越界，只要页面肉眼看起来过挤、信息难以扫读，也视为未通过

## D. 图片重页面

以下页面属于高风险图片页，必须重点检查：

- 架构图页
- 截图页
- 进展图 / 甘特图页
- 图文混排但图片承载主要信息的页面

重点确认：

- 图片展示尺寸足够大，核心信息能够肉眼辨认
- 图片没有被缩到“虽然存在但看不清内容”的程度
- 近方形架构图、流程图、截图不要塞进过宽且过矮的 `contain` 容器里
- 图片与标题、说明文字之间有足够留白
- 图片下方说明文字不会与其他模块重叠
- 如果图片是页面主要信息源，而当前尺寸下无法阅读，则该页视为未通过

## E. 导出后视觉复核

- 至少打开或渲染检查：
  - 1 页目录页
  - 1 页普通内容页
  - 1 页密集内容页
  - 1 页图片重页面
- 推荐先运行：

```bash
python3 scripts/render_svg_pages.py <project_path> -s final
```

- 再从渲染结果中抽检以上页面。
- 对“长亭通用墨绿色”（模板 ID：`chaitin`），额外确认：
  - Logo、页码、底部装饰条与正文没有打架
  - 目录页数字没有压住标题或副标题
  - takeaway 与正文层之间有明确间隔

## F. 字体与 QA 记录

执行完 `svg_to_pptx.py <project_path> -s final` 后：

1. 运行：

```bash
python3 scripts/check_pptx_fonts.py <project_path>
```

2. 确认没有字体替换、文字碰撞、图片不可读或视觉过挤问题。
3. 如果发现问题，返回修 SVG、字体策略、页面布局或模板资源，然后重新跑后处理和导出。
4. 最终交付前建议输出：

```bash
python3 scripts/write_qa_manifest.py <project_path> --format <format>
```

让本次 QA 结果可追溯。对于长亭相关模板，若 manifest 中的 `svg_output_quality` / `svg_final_quality` 显示未通过，则不得交付。
