# 安服 PPT 正文页布局提示词模板

> **模板**: chaitin_anfu（长亭安服）
> 从长亭安服主打胶片(33页) + HW总结(119页)共123页正文页中提取的高频布局模式。
> 用途：Strategist 在 Content Outline 中为每页指定 `layout_type`，Executor 按提示词描述生成 SVG，无需硬编码模板。

---

## 布局类型总览

|| # | layout_type | 占比 | 典型场景 |
|---|-------------|------|----------|
| 1 | **lr_split_imagetext** | **52%** | **左文右图（绝对主导布局）** |
| 2 | lr_split_balanced | 21% | 章节概述、服务模块说明 |
| 3 | lr_split_dense | 18% | 能力矩阵、多维度列表 |
| 4 | lr_split_righttitle | 10% | 总览页、右标题+左内容 |
| 5 | standard | 8% | 核心观点、单一重点 |
| 6 | tb_split | 8% | 上下对比、流程+说明 |
| 7 | table_page | 6% | 数据表、清单、对比表 |
| 8 | lr_split_lefttitle | 3% | 深度技术详解 |
| 9 | chart_page | 2% | 数据可视化、KPI仪表盘 |
| 10 | custom | — | 不适合以上任何类型的页面 |

> ⚠️ **lr_split_imagetext 占所有正文页的 50%+**，是安服PPT的绝对主导布局。Strategist 应优先考虑此类型。
> 注：standard_rich(2.4%)、dense_grid(0.8%)、tb_split_dense(0.8%) 合并到相近类型或 custom。

## 画布与安全区（所有类型共用）

- 画布：`0 0 1280 720`
- 页头区：y=26~90（白底标题栏 + 品牌色标题 + Logo）

**白底+品牌色风格（强制 — 与真实安服PPT一致）**：

> ⚠️ 基于2个真实案例蒸馏（HW总结119页 + 安服主打胶片33页）：**正文页统一白底**，暗色仅用于封面/章节/结尾页。

| 角色 | 色系A（品牌绿·默认） | 色系B（青绿·HW总结） | 说明 |
|------|----------------------|----------------------|------|
| 页面背景 | `#FFFFFF` | `#FFFFFF` | **正文页白底** |
| 主色/标题强调 | `#7BBD4A` | `#43827F` | 标题装饰条/强调 |
| 卡片背景 | `#F5F7FA` | `#F5F7FA` | 浅灰卡片底 |
| 正文深色 | `#1A1A1A` | `#1A1C1E` | 主文字 |
| 正文灰色 | `#404040` | `#404040` | 副文字 |
| 正文浅灰 | `#666666` | `#666666` | 辅助说明 |
| 标注/页码 | `#A6A6A6` | `#A6A6A6` | 最浅文字 |
| 分隔线/边框 | `#E0E0E0` | `#E0E0E0` | 分隔 |
| 告警红 | `#FF0000` | `#FF0000` | 高危/攻击标记 |
| 严重红 | `#C00000` | `#C00000` | 严重等级 |
| 警告黄 | `#FFFF00` | `#FFFF00` | 中危/关注 |

> ⚠️ **禁止正文页使用暗色背景**：`#1A1A2E`/`#222244` 等深色仅限封面/章节/结尾页。
> 白底+品牌色是真实安服PPT的标准风格，与用户预期一致。

- 内容安全区（body_safe_region）：x=34, y=90, w=1212, h=580 → 即 x=34~1246, y=90~670
- 页脚区：y=670~720（底部分隔线+页码）
- 字体栈：`"Microsoft YaHei", "PingFang SC", Arial, sans-serif`

## 🔴 图片强制规则（最高优先级）

> 基于2个真实案例蒸馏：**95%正文页含有图片**，纯文字页几乎为0。当前管线最大短板就是图片缺失。

**规则 1：正文页必须嵌入图片**
- 每个 body page 至少包含 1 张图片（`<image href="../images/xxx">`）
- 图片来源优先级：① 项目 `images/` 目录中的源文档素材 > ② 产品截图/架构图 > ③ SVG 自绘示意图
- 如果 `images/` 中无匹配图片，Executor 必须用 SVG 绘制示意图（流程图/架构图/数据图），不能只堆文字

**规则 2：图片位置 — 优先 img_right**
- 50%+的正文页使用 img_right（文字在左，图片在右）
- 默认布局假设图片在右栏，文字在左栏
- 仅在有充分理由时（如产品全屏截图）才用 img_left 或 img_bottom

**规则 3：图片尺寸 — 以小图为主**
- 91%的图片占页面面积 <10%（小截图/小图标/小数据图）
- 典型尺寸：宽 200~400px，高 120~250px
- 不要追求全幅大图；产品截图通常 300px 宽即可
- KPI 大数字/统计数据不在此规则内

**规则 4：图片类型匹配**

| 页面内容 | 推荐图片类型 | 典型尺寸(px) |
|----------|------------|-------------|
| 攻击路径/流程 | SVG 自绘流程图 | 500×300 |
| 产品功能 | 产品界面截图 | 350×200 |
| 数据统计 | SVG 柱状图/饼图 | 400×250 |
| 安全事件 | 时间线/趋势图 | 500×200 |
| 能力矩阵 | 图标网格(4~6个) | 每个60×60 |
| 架构概述 | 架构图/拓扑图 | 450×300 |
| 漏洞清单 | 告警截图/代码片段 | 300×180 |

**规则 5：SVG 图片引用**
- 使用 `<image href="../images/xxx.png" x="..." y="..." width="..." height="..."/>`
- 绝对不要用裸文件名或 `images/xxx.png` 前缀
- 如果图片不存在，用 SVG rect+text 绘制占位示意图，标注 `[示意图]`

## 1. lr_split_balanced（左右均衡分栏）

**占比**：21.1% — 最高频布局

**结构描述**：
左右两栏宽度接近（6:4 或 5:5），各自独立承载文字内容。左栏通常放主要论述，右栏放支撑要点或并列信息。两栏之间留 40~60px 间隔。

**坐标规范**（基于 body_safe_region x=34 y=90 w=1212 h=580）：
```
左栏：x=34, y=100, width=586, height=550
右栏：x=660, y=100, width=586, height=550
间隔：40px（660-34-586=40）
```

**字号规范**：
| 元素 | 字号 | 颜色 |
|------|------|------|
| 栏标题 | 22px medium | `#1A1A1A` |
| 正文 | 16px regular | `#333333` |
| 要点条目 | 15px regular | `#666666` |
| 栏标题绿色装饰线 | 3px高 | `#7BBD4A` |

**适用场景**：
- 章节概述（左=服务范围，右=核心优势）
- 双维度对比（左=现状，右=目标）
- 并列模块说明

**Strategist 指定示例**：
```
P08: layout_type=lr_split_balanced
  左栏: "攻击面管理服务范围" — 5个要点
  右栏: "核心能力优势" — 4个要点+绿色高亮关键词
```

**Executor 生成要点**：
- 两栏高度应对齐，视觉平衡
- 每栏内部可用卡片/色块包裹，卡片背景 `#F5F7FA`（浅绿底）
- 栏标题前加 3px 绿色短竖线或短横线装饰
- 若某栏要点超过6条，考虑拆为2列或换用 lr_split_dense
- **视觉锚点**：两栏结构对称，无明显深色块或大图——这是与 imagetext/righttitle/lefttitle 的核心区别。两栏顶部各有一条 `section_label_green` 绿色标签栏，内部用 `content_card` 卡片堆叠。

## 1. lr_split_imagetext（左文右图 — 绝对主导布局）

**占比**：52% — **安服PPT最高频布局，每2页就有1页是此类型**

**结构描述**：
左栏以文字说明为主，右栏以图片/图示为主（img_right 占此类型的 95%+）。图片侧可放 1 张大图+标注，或 2~3 张小图竖排。文字侧配合图片内容做要点说明。这是安服PPT的"默认布局"，其他布局类型只在内容不适合左文右图时才使用。

**坐标规范**（基于 body_safe_region x=34 y=90 w=1212 h=580）：
```
文字栏（左）: x=34, y=100, width=546, height=550
图片栏（右）: x=610, y=100, width=636, height=550
或反向（少见）: 图片栏（左）: x=34, y=100, width=636, height=550
              文字栏（右）: x=690, y=100, width=556, height=550
```

**字号规范**：
| 元素 | 字号 | 颜色 |
|------|------|------|
| 文字栏标题 | 22px medium | 品牌主色 |
| 正文 | 16px regular | `#404040` |
| 图片说明/标注 | 13px regular | `#666666` |
| 图片编号标签 | 14px bold, 白色文字+品牌色底 | 品牌主色背景 |

**图片嵌入规范**（强制）：
- 右栏**必须**包含至少 1 个 `<image>` 标签或 SVG 自绘示意图
- 优先使用项目 `images/` 中的源文档素材：`<image href="../images/xxx.png" x="610" y="120" width="300" height="200"/>`
- 多图竖排时：图间距 20px，每图高度 120~180px，宽度统一 300~400px
- 如果无匹配素材，用 SVG 绘制：流程图(rect+polygon箭头)/架构图(层次方块)/数据图(bar/line)
- 图片下方留 20px 给说明文字

**适用场景**：
- 安全案例展示（左=案例描述，右=攻击链路图/截图）— **最常见**
- 产品功能图解（左=功能列表，右=产品截图）
- 技术原理说明（左=流程说明，右=拓扑图）
- 攻防路径分析（左=攻击步骤，右=路径图）

**Strategist 指定示例**：
```
P15: layout_type=lr_split_imagetext
  左栏文字: "XX攻防案例 — 攻击路径分析" — 4步流程
  右栏图片: 攻击链路示意图(../images/attack_path.png)，3个关键节点标注
```

**Executor 生成要点**：
- **图片优先放在右栏**（符合从左到右阅读流：先读文再看图）— 这是 img_right 占 95%+ 的原因
- 若图片是流程图/架构图且无位图素材，用 SVG 矩形+箭头直接绘制
- 图片下方留 20px 给说明文字
- 多图时竖向排列，图间距 20px，每图高度不超过 120px
- **视觉锚点**：右栏必须有视觉图示（`<image>` 或 SVG 绘图），文字侧无大图。图片区占位高度≥栏高60%。

## 3. lr_split_dense（左右密排分栏）

**占比**：17.9%

**结构描述**：
左右两栏都承载密集文字内容，总 shape 数≥15。通常每栏内再分2~3个子卡片或色块，形成矩阵感。是安服 PPT 中"能力清单""技术指标""服务项总览"的典型布局。

**坐标规范**（基于 body_safe_region x=34 y=90 w=1212 h=580）：
```
左栏: x=34, y=100, width=586, height=550
右栏: x=660, y=100, width=586, height=550
子卡片：宽≈566，高≈120~140，间距 16px
```

**字号规范**：
| 元素 | 字号 | 颜色 |
|------|------|------|
| 卡片标题 | 18px medium | `#1A1A1A` |
| 卡片正文 | 14px regular | `#333333` |
| 标签/徽章 | 12px bold | 白色+`#7BBD4A`底 |
| 序号 | 16px bold | `#7BBD4A` |

**适用场景**：
- 安全能力矩阵（6~8项能力分两栏排列）
- 服务项清单（每项一行简述）
- 技术指标总览（多维数据条目）

**Strategist 指定示例**：
```
P12: layout_type=lr_split_dense
  左栏: "检测能力" — 4项（流量分析、威胁情报、行为检测、异常发现）
  右栏: "响应能力" — 4项（自动阻断、溯源分析、工单流转、复测验证）
```

**Executor 生成要点**：
- 每个子卡片必须用圆角矩形（rx=8），背景 `#F5F7FA`
- 卡片左上角放绿色序号或图标，标题紧跟其后
- 正文用 `<tspan>` 换行，每行不超过 24 字
- 如果条目超过8个（每栏4个），降低字号到 13px 或拆分到下一页
- 条目间用 1px `#E0E0E0` 分隔线比用间距更紧凑
- **视觉锚点**：两栏内均密排3~4个子卡片（`grid_card` + `number_circle_filled` 序号），总 shape≥15。视觉密度显著高于 balanced（balanced 每栏只有1~2个色块）。子卡片紧凑堆叠，无大图无深色块。

## 4. lr_split_righttitle（右标题+左内容）

**占比**：9.8%

**结构描述**：
标题/大字在右侧，内容主体在左侧。与常规"左标题右内容"相反，用于总览性页面——右侧大字点明主题，左侧展开具体内容。

**坐标规范**（基于 body_safe_region x=34 y=90 w=1212 h=580）：
```
内容栏（左）: x=34, y=100, width=736, height=550
标题栏（右）: x=800, y=100, width=446, height=550
  大标题: x=800, y=310, width=446, 字号 36~48px
  副标题/说明: x=800, y=370, width=446, 字号 16px
```

**字号规范**：
| 元素 | 字号 | 颜色 |
|------|------|------|
| 右侧大标题 | 36~48px bold | `#1A1A1A` 或 `#7BBD4A` |
| 右侧副标题 | 16px regular | `#666666` |
| 左侧栏标题 | 20px medium | `#1A1A1A` |
| 左侧正文 | 15px regular | `#333333` |

**适用场景**：
- 章节总览页（右=章节名，左=本章涵盖内容）
- 服务全景页（右=服务大类名，左=子项列表）
- 方法论概述（右=方法论名，左=步骤拆解）

**Strategist 指定示例**：
```
P05: layout_type=lr_split_righttitle
  右侧: "全流量分析" (大标题)
  左侧: 3个要点 — 数据采集方式、分析引擎、告警输出
```

**Executor 生成要点**：
- 右侧标题区可用深色背景块（`#000000`）+白色/绿色文字形成视觉锚点
- 左侧内容用卡片或列表排列
- 右侧标题区底部加绿色短横线装饰
- 若右侧需放图标，放在标题上方，尺寸 48~64px
- **视觉锚点**：右侧必须有深色背景块（`#000000` rect 占右侧区域100%宽度+高度），块内大标题用 `#FFFFFF` 或 `#7BBD4A`。这是与 balanced/imagetext 的核心区别——只有 righttitle 允许大块深色区域。无 `<image>` 标签。

## 5. standard（标准单栏）

**占比**：8.1%

**结构描述**：
单一核心内容区，不左右分栏。适合表达一个核心观点、一张关键图表、或3~5个并列要点。是"只说一件事"的页面。

**坐标规范**（基于 body_safe_region x=34 y=90 w=1212 h=580）：
```
内容区: x=34, y=100, width=1212, height=550
居中标题: x=640, y=130, text-anchor=middle
要点区: x=34~1246, y=190~640
```

**字号规范**：
| 元素 | 字号 | 颜色 |
|------|------|------|
| 页标题 | 30~34px medium | `#1A1A1A` |
| 核心观点/结论 | 24px bold | `#1A1A1A` |
| 正文 | 18px regular | `#333333` |
| 注释 | 14px regular | `#666666` |

**适用场景**：
- 核心结论页
- 安全理念/原则
- 单一重点强调（如"零信任架构"）

**Strategist 指定示例**：
```
P03: layout_type=standard
  核心观点: "安全运营的本质是持续对抗"
  3个支撑论点
```

**Executor 生成要点**：
- 内容区可居中排列，也可左对齐
- 3~5个并列要点可用等宽卡片横排（每卡片宽≈220px，间距 20px）
- 页面留白比其他类型更多，呼吸感强
- 核心观点可用绿色底色条（`#7BBD4A`底+白色字）突出

## 6. tb_split（上下分栏）

**占比**：8.1%

**结构描述**：
上半区放流程/架构/图表，下半区放文字说明。或上半区放文字概述，下半区放详细条目。两区之间用分隔线或色带区分。

**坐标规范**（基于 body_safe_region x=34 y=90 w=1212 h=580）：
```
上半区: x=34, y=100, width=1212, height=260
下半区: x=34, y=390, width=1212, height=260
分隔线/色带: y=370, height=4px, fill=#7BBD4A
```

**字号规范**：
| 元素 | 字号 | 颜色 |
|------|------|------|
| 上区标题 | 22px medium | `#1A1A1A` |
| 上区正文 | 16px regular | `#333333` |
| 下区条目标题 | 18px medium | `#1A1A1A` |
| 下区条目正文 | 15px regular | `#666666` |

**适用场景**：
- 上=流程图/架构图，下=各环节说明
- 上=核心数据/KPI，下=解读
- 上=原则/方法，下=具体措施

**Strategist 指定示例**：
```
P20: layout_type=tb_split
  上区: 应急响应5阶段流程（发现→确认→遏制→清除→恢复）
  下区: 每阶段的关键动作与交付物
```

**Executor 生成要点**：
- 上区若画流程图，节点用圆角矩形（rx=6），箭头用 `<polygon>` 三角
- 流程节点间距均匀，每节点宽≈160px
- 下区用左对齐列表或2列卡片
- 上下区分隔用绿色细线（2~4px），不要用粗色块割裂页面

## 7. table_page（数据表格页）

**占比**：5.7%

**结构描述**：
以表格为核心内容，展示数据对比、清单、或结构化信息。表头用品牌色，数据行交替底色。

**坐标规范**（基于 body_safe_region x=34 y=90 w=1212 h=580）：
```
表格区: x=34, y=120, width=1212, height=520
表头行高: 44px, 背景 #7BBD4A, 文字白色
数据行高: 36px, 奇数行 #FFFFFF, 偶数行 #F5F7FA
列数: 3~6列，首列宽占比约 25%
```

**字号规范**：
| 元素 | 字号 | 颜色 |
|------|------|------|
| 表头文字 | 15px bold | `#FFFFFF` |
| 数据文字 | 14px regular | `#333333` |
| 表格标题 | 22px medium | `#1A1A1A` |
| 表注 | 12px regular | `#666666` |

**适用场景**：
- 漏洞清单（漏洞名/等级/影响/修复状态）
- 服务对比表
- 资产清单/检查项

**Strategist 指定示例**：
```
P25: layout_type=table_page
  表格: "高危漏洞修复进度" — 5列(漏洞名/CVE/等级/影响系统/状态)
  约8行数据
```

**Executor 生成要点**：
- 用 `<rect>` 画每个单元格，不要用 `<line>` 画网格线（导出兼容性差）

- 状态列可用色标：绿色=已完成，黄色=进行中，红色=未处理
- 超过10行时分页，不要压缩行高

## 8. lr_split_lefttitle（左标题+右内容）

**占比**：3.3%

**结构描述**：

**布局选择决策树**：

```
内容特征？
├── 不适合以下任何类型 → custom（自由布局，需在 design_spec 中详细描述）
├── 纯数据/表格 → table_page

    ├── 标题在左侧 → lr_split_lefttitle
```

- 同一章内至少出现2种不同布局

- 任何正文页 layout_type 都必须包含至少 1 张图片或 SVG 示意图

## 安服套路提示词模板（L3层 — 高层排版指令）

> **定位**：L3 层是效果最大的单点——它告诉 LLM "这页应该长什么样"，而不是只给坐标参数。
> 每个套路=布局模式名 + 结构描述 + 组件规则 + 颜色语义 + 适用场景。
> Strategist 在 Content Outline 中为页面指定 `routine_id`，Executor 按对应套路生成。

### 套路使用规则

1. **匹配原则**：Strategist 根据页面内容语义自动匹配最接近的套路，写入 design_spec
2. **降级原则**：如果没有匹配的套路，使用布局决策树选择 layout_type，不强制匹配套路
3. **组合原则**：一个套路可对应多种 layout_type（如 sec-attack-chain 可用 lr_split_imagetext 或 tb_split）
4. **优先级**：套路指令 > 坐标规范 > 字号规范（套路层覆盖 L1/L2 层的默认值）

### 匹配语法（Strategist → Content Outline）

在 design_spec.md 的 IX. Content Outline 中，为页面同时指定 layout_type 和 routine_id：

```markdown
### P08 攻击路径分析
- layout_type: lr_split_imagetext
- routine_id: sec-attack-chain
- 时间轴节点: 边界突破→内网横移→权限提升→数据窃取
- 详情卡片: 对应节点2和3的技术细节
- 图片: attack_path_diagram.svg（自绘攻击链流程图）
```

如果内容匹配多个套路，选最核心的一个——一页只能有一个 routine_id。
