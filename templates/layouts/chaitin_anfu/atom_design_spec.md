# chaitin_anfu 原子化正文页架构设计方案

> 本文档定义 chaitin_anfu 模板正文页的原子化组件体系，
> 目标是从"1种通用壳"进化为"可动态组合的原子组件系统"，
> 在保证品牌一致性的同时，避免模板化同质化。

---

## 一、问题定义

**现状**：
- `03_content.svg` 只有一层 `{{CONTENT_AREA}}`，所有正文页共用同一个壳
- Executor 把所有内容塞进一个 GROUP 容器
- 后果：25页 PPT 所有正文页结构完全相同，只有文字不同

**目标**：
- 建立原子组件库，让正文页能够根据内容语义动态组合出不同结构
- 保证品牌一致性（颜色/字体/尺寸全部写死）
- 把不稳定性锁在固定模板层，不让生成器自由发挥

---

## 二、四层稳定性架构

```
┌─────────────────────────────────────────────────────────────┐
│ 第一层：固定模板层（零生成风险）                               │
│   封面/目录/章节页/结尾页  →  100%固定，不参与动态生成         │
├─────────────────────────────────────────────────────────────┤
│ 第二层：原子组件层（稳定区）                                   │
│   30种原子组件，颜色/尺寸全部写死                              │
│   → 生成器只控制：内容文字 + 位置坐标 + 数量                   │
├─────────────────────────────────────────────────────────────┤
│ 第三层：组合模板层（较不稳定区）                               │
│   10种组合规则，约束原子之间的排布关系                         │
│   → 超过3个原子组合时触发碰撞检测                             │
├─────────────────────────────────────────────────────────────┤
│ 第四层：高阶固定模板层（黑盒区）                               │
│   6个预制高阶页面（时间线/矩阵/流程/公式等）                   │
│   → 按内容关键字匹配，整页替换文字                            │
└─────────────────────────────────────────────────────────────┘
```

## 六、布局约束检查器规则

> 生成后必须通过以下检查，否则不允许导出。

### 规则1：碰撞检测

任何两个原子组件的边界不得重叠（允许刚好相切）。

### 规则2：容量约束

| 原子类型 | 最大字符数 |
|---------|-----------|
| subtitle_tag | ≤8字符 |
| data_highlight数字 | ≤5字符 |
| data_block说明 | ≤20字符 |
| number_box | 01~99 |
| tag_pill | ≤6字符 |
| section_label | ≤10字符 |

### 规则3：品牌颜色锁定

生成器不得动态指定以下颜色，必须使用模板中预定义值：

| 用途 | 颜色值 |
|------|-------|
| 品牌绿 | `#7BBD4A`, `#65C133`, `#77BD5E`, `#75BC60`, `#A7D18E` |
| 蓝绿辅助 | `#43827F`, `#44887F` |
| 数据高亮 | `#FFFF00` |
| 边框/分割线 | `#E0E0E0`, `#A6D7BA`, `#A9D18E` |
| 页码灰 | `#A6A6A6` |
