# 长亭安服重型页 SVG 骨架蓝图

> 用途：把 `complex_graph_semantics.md` 中的抽象语义，进一步落到 `security_service` 的具体 SVG 承载骨架上。
>
> 目标不是把模板锁死，而是给高价值复杂页提供一套更稳的默认承载框架：固定骨架负责稳定，正文内容区负责灵活。

---

## 一、适用范围

本文件当前重点约束以下三类重型正文页：

- `19_result_leading_case.svg`：攻击链 / 复杂案例链 / 结果证明页
- `07_data.svg`：证据证明页 / 数据 + 证据混合证明页
- `16_table.svg`：治理优先级矩阵 / 控制矩阵 / 闭环整改页

---

## 二、攻击链重型页蓝图

对应页型：`19_result_leading_case.svg`

### 页面任务

- 把“入口 - 动作 - 放大 - 结果”压成一条可读的主链
- 让证据和主链节点形成明确附着，而不是散落在页面边缘
- 在页尾给出一句管理判断，而不是停在过程描述

### 推荐阅读路径

```text
第一眼：结果 headline / 主判断
第二眼：主链四段结构
第三眼：节点证据与结果证据
第四眼：页尾管理收束
```

### 默认结构分区

- `headline-zone`：结果 headline + 场景定位
- `metric-zone`：关键结果指标或单个强结果值
- `case-chain-zone`：四段主链，含入口节点 / 动作节点 / 放大节点 / 结果节点
- `evidence-rail-zone`：直接证据 / 结果证据 / 管理判断侧栏
- `closure-band-zone`：影响判断 / 动作要求 / 下一页承接

### 默认语义分工

- 主链节点：只写对象 + 动作 / 状态变化
- 证据侧栏：只写“该图证明什么”
- 页尾收束：只写管理判断，不复述主链

### 禁止退化

- 三列并排挑战 / 动作 / 结果，缺少主链语义
- 证据图没有对应节点锚点
- 结果只放在标题，不进入结构终点

---

## 三、证据证明页蓝图

对应页型：`07_data.svg`

### 页面任务

- 把 KPI、结构图、证据图、收束结论压成同一页证明体系
- 让数据证明“规模 / 程度”，让证据证明“真实性 / 可复现性”

### 推荐阅读路径

```text
第一眼：证明 headline
第二眼：KPI 锚点
第三眼：主证明结构图
第四眼：证据栈与结论带
```

### 默认结构分区

- `headline-zone`：证明目标 / 结果 headline / 管理语境
- `kpi-zone`：2-3 个指标锚点，不宜过多
- `proof-zone`：现象 / 核心证明 / 结果指向的主结构图
- `evidence-stack-zone`：直接证据 / 结果证据 / 判断收束
- `closure-belt-zone`：证明结论 / 风险意义 / 动作要求

### 默认语义分工

- KPI：说明规模与程度
- 主证明图：说明关系与结构
- 证据栈：说明哪些截图在证明哪个节点或结果
- 收束带：说明管理层该如何理解本页

### 禁止退化

- 做成“4 个 KPI + 一张普通图表”
- 证据图和指标图没有分工
- 侧栏只写描述，没有判断句

---

## 四、治理矩阵页蓝图

对应页型：`16_table.svg`

### 页面任务

- 把问题翻译成优先级、控制动作、验证方式和闭环要求
- 让读者一眼看到先做什么、怎么验、做到什么算完成

### 推荐阅读路径

```text
第一眼：优先级判断
第二眼：域 - 状态 - 动作 - 验证矩阵
第三眼：闭环步骤
第四眼：右侧管理决策栏
```

### 默认结构分区

- `priority-zone`：P1 / P2 / P3 顶部优先级卡
- `matrix-zone`：优先级 / 治理域 / 当前状态 / 关键动作 / 验证闭环矩阵
- `closure-zone`：收口 / 复测 / 留痕三段闭环带
- `decision-rail-zone`：管理判断 / 关键约束 / 完成标准 / 资源要求

### 默认语义分工

- 顶部优先级卡：讲排序逻辑
- 矩阵主体：讲域、动作、验证映射
- 闭环带：讲动作之后如何形成可复核状态
- 决策侧栏：讲管理层需要推动什么

### 禁止退化

- 纯表格清单，没有优先级主线
- 只有动作，没有验证和完成标准
- 右侧侧栏只是重复矩阵内容

---

## 五、使用规则

命中上述三类重型页时，建议执行顺序固定为：

```text
先完成复杂页建模
→ 再命中本蓝图的分区结构
→ 再把节点 / 证据 / 收束语句填进对应区域
→ 最后做文本适配与视觉 QA
```

如果原文内容不足以支撑这些分区，就不应强行套用重型骨架，而应回退到更轻的页型。
