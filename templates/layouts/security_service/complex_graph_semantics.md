# 长亭安服复杂图形语义规则

> 用途：把安服复杂页从“结构化排版”提升为“结构化建模”。  
> 这份文档不定义具体某一页长什么样，而定义复杂页里的节点类型、关系类型、阅读主线与视觉职责。

> 与具体 SVG 承载骨架的映射关系，见 `complex_svg_blueprints.md`。

---

## 一、目标

安服复杂页的关键，不是“画很多节点”，而是把安服语义稳定转成图形语法。

后续所有复杂页，都应优先回答：

- 这页的主判断是什么
- 需要哪些节点类型来承载这个判断
- 节点之间是什么关系
- 哪些信息画在图里，哪些信息写在图外

---

## 二、复杂图形的三层建模

### 2.1 结构层

结构层回答：

- 主轴是什么
- 分层还是分域
- 是链路、矩阵、闭环还是泳道

结构层决定整页的图形骨架。

### 2.2 语义层

语义层回答：

- 每个节点在安服语境里扮演什么角色
- 每条连线到底表示因果、依赖还是放大

语义层决定图形不是“装饰连接”，而是“可解释连接”。

### 2.3 证据层

证据层回答：

- 哪些节点需要挂原始截图、数据、日志、图片
- 哪些结论是直接证据支撑
- 哪些只是背景说明

证据层决定图不是空结构，而是可验证的结构。

---

## 三、节点类型库

复杂页原则上优先从下面节点类型中选择，而不是把所有节点都画成同一种卡片。

### 3.1 入口节点

表示：

- 漏洞入口
- 暴露面
- 人员入口
- 供应链入口
- 管理面入口

适用语义：

- 初始触点
- 第一落点来源

推荐表现：

- 放在链路起点
- 颜色可偏蓝 / 青
- 文字尽量短，强调对象与方式

推荐字段：

- 入口对象
- 利用方式
- 代表性实例

---

### 3.2 动作节点

表示：

- 利用动作
- 攻击动作
- 执行动作
- 横向动作
- 控制动作

适用语义：

- 攻击方做了什么
- 机制如何推进

推荐表现：

- 放在主链中段
- 颜色可偏浅蓝 / 中蓝
- 节点名优先写动作，不只写对象

推荐字段：

- 关键动作
- 利用方式
- 结果变化

---

### 3.3 放大节点

表示：

- 配置失陷
- 凭证泄露
- 管理面暴露
- 通用口令
- 敏感信息外泄

适用语义：

- 为什么攻击能继续走下去
- 为什么某个问题价值被放大

推荐表现：

- 常作为侧向条件或中间增益块
- 可用橙色/强调色
- 应与主动作链形成“增强关系”

推荐字段：

- 放大条件
- 触发影响
- 对主链的贡献

---

### 3.4 结果节点

表示：

- 控制结果
- 权限结果
- 数据结果
- 云侧结果
- 业务影响

适用语义：

- 攻击最终拿到了什么
- 对业务意味着什么

推荐表现：

- 放在链路终段或右侧收束区
- 高亮颜色应更明确
- 要求有结论感

推荐字段：

- 结果对象
- 控制范围
- 风险后果

---

### 3.5 证据节点

表示：

- 原始截图
- 日志片段
- 控制台画面
- 指标图
- 外部旁证

适用语义：

- 支撑某一节点或某一判断

推荐表现：

- 不建议与主结构节点同权
- 更适合作为挂载块、侧栏块、节点附着块
- 必须配一句“该图证明什么”

推荐字段：

- 证据类型
- 支撑对象
- 证明结论

---

### 3.6 判断节点

表示：

- 管理判断
- 风险判断
- 结论判断
- 策略判断

适用语义：

- 这页最终想告诉读者什么

推荐表现：

- 放在页头、页尾或右侧总结区
- 不和普通说明块混用
- 文字应像顾问判断，而不是描述

推荐字段：

- 主判断
- 分判断
- 建议动作

---

### 3.7 控制节点

表示：

- 治理动作
- 整改动作
- 验证动作
- 闭环动作

适用语义：

- 下一步应该做什么
- 哪些动作构成闭环

推荐表现：

- 适合出现在治理页、整改页、闭环页
- 常与箭头、阶段带、循环带配合

推荐字段：

- 动作
- 验证方式
- 完成标准

---

## 四、关系类型库

复杂图的连线不能只表示“连起来了”，必须尽量表达关系类型。

### 4.1 因果关系

含义：

- A 导致 B
- A 成立后才出现 B

适用：

- 攻击链主链
- 结果链

图形建议：

- 实线主箭头
- 主路径上优先使用

### 4.2 依赖关系

含义：

- B 的成立依赖 A
- 没有 A，B 不成立

适用：

- 条件页
- 配置与权限页

图形建议：

- 细线 / 辅助箭头
- 不要和主因果线完全同权

### 4.3 放大关系

含义：

- A 本身不是主路径，但会显著提升 B 的利用价值

适用：

- 凭证泄露
- 管理面暴露
- 通用口令

图形建议：

- 斜向连接
- 强调色
- 可从侧边条件块连接到主链

### 4.4 并行关系

含义：

- 多条路径同时存在
- 多个入口并行出现

适用：

- 多入口攻击面
- 多角色协同

图形建议：

- 并列结构
- 汇聚前保持独立

### 4.5 汇聚关系

含义：

- 多个入口最终汇聚到同一控制面或结果面

适用：

- 风险总览页
- 攻击树

图形建议：

- 多支线汇入一个主结果块
- 强调“不是孤立问题”

### 4.6 对照关系

含义：

- 两类对象/状态/策略的差异比较

适用：

- 治理对照
- 前后差异
- 客户侧 / 长亭侧

图形建议：

- 左右列
- 对照块应保持同维度

### 4.7 反馈 / 闭环关系

含义：

- 某一步结果会回流到下一轮治理

适用：

- 运营闭环
- 复测闭环

图形建议：

- 回环箭头
- 不宜只画成单向流程

## 五、安服复杂页的典型结构骨架

### 5.1 主链 + 条件 + 结果

适合：

- 攻击链
- 案例链

结构：

```text
入口 → 动作 → 控制
   ↘ 放大条件 ↗
            → 结果
```

### 5.2 分层 + 汇聚

适合：

- 攻击树
- 风险结构页

结构：

```text
入口层
→ 利用层
→ 放大层
→ 目标/结果层
```

### 5.3 主图 + 证据挂载

适合：

- 证据证明页
- 重型案例页

结构：

```text
主结构图
+ 右侧/下方证据块
+ 总判断
```

### 5.4 控制矩阵 + 闭环带

适合：

- 整改页
- 治理页

结构：

```text
域 / 状态 / 动作 / 验证
+ 底部闭环流程
+ 右侧管理判断
```

## 六、图文分工规则

复杂页禁止图文职责混乱。

推荐分工：

- 图负责：
  - 层级
  - 路径
  - 关系
  - 证据挂位
- 文负责：
  - 判断
  - 解释
  - 风险意义
  - 建议动作

禁止情况：

- 图里写满解释句
- 图外再重复一遍节点名
- 原图只是配图，不承担证明作用

---

## 七、当前与案例的主要差距提醒

在生成时，应重点避免以下退化：

- 只做事件顺序，不做机制关系
- 所有节点都长得一样，看不出角色差异
- 证据图只是配图，不是证据锚点
- 有结构，但主判断不明确
- 信息很多，但第一眼抓不到主线

若出现这些情况，说明页面仍停留在“复杂排版页”，还没有进入“复杂知识表达页”。

---

## 八、重型页骨架映射

当复杂页不只是“要有结构”，而是要承载案例级高密度论证时，建议优先使用以下骨架：

- 攻击链 / 复杂案例链 / 结果证明：优先命中 `19_result_leading_case.svg`
  - 目标结构：`headline + 四段主链 + 证据侧栏 + 收束带`
- 证据证明页 / 数据 + 证据混合页：优先命中 `07_data.svg`
  - 目标结构：`证明 headline + KPI 锚点 + 主证明结构图 + 证据栈 + 结论带`
- 治理矩阵 / 优先级整改页：优先命中 `16_table.svg`
  - 目标结构：`优先级卡 + 域状态动作验证矩阵 + 闭环带 + 决策侧栏`

具体分区与阅读路径，统一见 `complex_svg_blueprints.md`。
