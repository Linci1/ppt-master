# 长亭安服复杂页推理模板

> 用途：作为 `Step 6.0 复杂页建模` 的标准输出模板。  
> 目标：让复杂页在开始画 SVG 前，先完成“页面角色 → 主判断 → 结构选择 → 节点关系 → 证据挂载 → 收束方式”的推理。

---

## 一、适用范围

以下页面默认使用本模板：

- 攻击链页
- 复杂案例链页
- 证据证明页
- 控制矩阵 / 治理看板页
- 闭环运营页
- 分层攻击树 / 风险结构页
- 多泳道实施链页

## 三、字段解释

### 3.1 页面角色

页面角色只能选一个：

- `概览页`
- `推进页`
- `证明页`
- `收束页`

说明：

- 概览页：负责建立总体认知
- 推进页：负责把论证往下推一层
- 证明页：负责拿出关键结构或证据
- 收束页：负责把判断导向整改、优先级或行动

禁止一个复杂页同时承担全部角色。

### 3.2 主判断

要求：

- 必须唯一
- 必须高于事实层
- 必须能驱动图形结构选择

示例：

- `互联网侧多个问题已汇聚成稳定突破链`
- `本次社工结果已足以证明人员入口可转化为办公网执行点`
- `治理重点不在单点清理，而在入口、凭证与管理面的同步收口`

### 3.3 分判断

要求：

- 最多 3 个
- 每个分判断都要支撑主判断
- 不允许和主判断平级

### 3.4 论证主线

复杂页必须先写出一条完整论证线，而不是边画边想。

推荐固定为：

- 现象 / 入口
- 放大机制 / 关键条件
- 结果 / 影响
- 管理判断 / 动作要求

要求：

- 标题、节点、证据说明、页尾判断必须共享这条主线
- 论证主线若写不清，说明这页还不适合进入复杂模式

### 3.5 主结构类型

从以下类型中选择：

- `链路`
- `分层`
- `矩阵`
- `闭环`
- `泳道`
- `证据挂载`
- `混合结构（必须说明原因）`

### 3.6 结构选择理由

必须回答：

- 为什么这一页适合用这个结构
- 这个结构比其他结构更适合表达什么

例如：

- `本页重点不是时间顺序，而是入口如何汇聚到同一结果，因此适合分层攻击树，不适合普通时间线`

### 3.7 为什么不用其他结构

强制做一次排除式思考，避免页型选错。

例如：

- `不选列表页，因为列表页无法表达入口、放大条件与结果的因果关系`
- `不选普通时间线，因为本页重点不是阶段推进，而是多入口汇聚`

### 3.8 证据分级与文本压缩计划

复杂页不仅要知道“证据挂在哪里”，还要知道“什么证据该占主位、什么文案该如何压缩”。

要求：

- `直接证据`、`结果证据`、`旁证 / 背景证据` 必须区分主次
- `标题句` 必须是判断句
- `节点文案` 必须压缩成对象 + 动作 / 状态变化
- `证据说明句` 必须直接回答“该图证明什么”
- `页尾收束句` 必须给出管理判断或动作要求

### 3.9 视觉焦点排序

复杂页必须明确第一眼到第三眼的阅读顺序。

推荐格式：

- 第一眼：主判断 / headline
- 第二眼：主结构图
- 第三眼：关键证据或结果区

### 3.10 页面收束方式

复杂页必须至少有一种收束：

- 管理判断
- 风险判断
- 建议动作

没有收束方式，不允许视为完成。

---

## 四、推理质量门

如果出现以下情况，说明推理还不够：

- 主判断只是事实总结，不是高层判断
- 分判断太多，已经失去主线
- 结构选择理由说不清
- 论证主线说不清，标题与页尾判断不在同一条线上
- 节点与关系列得出来，但不知道视觉主线怎么排
- 证据图存在，但说不清挂在哪里

这时不得直接进入 SVG 生成。

## 五、推荐示例

```text
页面角色：
证明页

页面意图：
证明互联网侧问题不是离散漏洞，而是已形成稳定突破链

证明目标：
让读者看到“入口 → 放大 → 结果”的结构性关系

主判断：
互联网侧多个问题已汇聚成稳定突破链

分判断：
1. 多个外网入口都可形成初始落点
2. 管理面暴露和凭证泄露显著放大利用价值
3. 风险结果已从漏洞发现转化为控制结果

论证主线：
- 现象 / 入口：多个互联网侧入口均可形成初始落点
- 放大机制 / 关键条件：管理面暴露与凭证泄露显著降低继续放大成本
- 结果 / 影响：风险已从漏洞发现转化为控制结果与云侧利用可能
- 管理判断 / 动作要求：治理重点应从单点修补转向入口与放大条件同步收口

主结构类型：
分层 + 汇聚

结构选择理由：
本页重点是展示不同入口如何汇聚到同一结果面，分层攻击树比时间线更合适

为什么不用其他结构：
不选普通列表页，因为列表无法表达层级、汇聚和放大关系

关键节点：
- 入口节点：ThinkPHP、Log4j2、后台暴露
- 动作节点：命令执行、后台访问、配置读取
- 放大条件：Nacos 暴露、凭证泄露、通用口令
- 结果节点：外网落点、管理面接管、云侧控制
- 证据节点：后台截图、凭证截图
- 判断节点：主风险判断

关键关系：
- 主链因果：入口 → 动作 → 控制结果
- 放大关系：管理面暴露 / 凭证泄露 → 提升主链价值
- 并行 / 汇聚：多个入口汇聚到同一结果面

证据挂载计划：
- 后台截图 → 管理面接管
- 凭证截图 → 云侧控制

证据分级：
- 直接证据：后台截图
- 结果证据：云侧控制结果图
- 旁证 / 背景证据：资产环境信息

文本压缩计划：
- 标题句：互联网侧多个问题已汇聚成稳定突破链
- 节点文案：统一压缩为对象 + 动作 / 状态变化
- 证据说明句：每张图都回答“该图证明什么”
- 页尾收束句：治理重点应从单点修补转为入口与放大条件同步收口

视觉焦点排序：
1. 主判断
2. 分层攻击树
3. 结果层与证据挂点

页面收束方式：
- 管理判断：当前风险已不是单点漏洞，而是结构性突破链
- 建议动作：先收口主入口，再同步消除放大条件
```
