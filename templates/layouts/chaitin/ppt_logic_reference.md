# 长亭通用墨绿色逻辑参考

> 目标：让 `chaitin` 模板在保持长亭品牌骨架稳定的前提下，输出更稳定的通用品牌型正文逻辑；既保留 AI 在内容区的灵活性，也避免所有页面退化成同一种“换皮卡片页”。

> 生成与校验时，还应同时参考：
> - `qa_profile.md`：品牌骨架与版式护栏
> - `text_prompt_snippets.md`：通用品牌正文页的可复用提示片段
> - `generation_checklist.md`：生成前、生成中、导出前检查清单

---

## 一、模板定位

`chaitin` 是 **长亭通用品牌表达模板**，不是安服能力证明型胶片，也不是固定话术的单一行业模板。

### 1.1 适用场景

- 企业内训
- 产品发布 / 方案介绍
- 技术分享 / 能力宣讲
- 品牌介绍 / 业务概览
- 混合型专题汇报（品牌 + 产品 + 方法论 + 案例）

### 1.2 不适合的场景

- 必须严格对齐历史胶片章节节奏的安服证明材料
- 需要大量浅底证据墙、资质墙、结果证明链的专项胶片
- 以正式政务规范或学术论文结构为主的汇报

---

## 二、推荐叙事节奏

`chaitin` 更像“品牌化通用底盘”，章节组织应优先服务主题表达，而不是强制套固定业务骨架。

### 2.1 常见章节骨架

```text
01 开场 / 背景
02 目标 / 挑战 / 现状
03 核心观点 / 方法论 / 能力结构
04 重点模块展开
05 案例 / 数据 / 价值证明
06 行动建议 / 收束
```

### 2.2 章节组织原则

- **先建立主题认知**：让观众先知道这份 PPT 在讲什么
- **再展开结构化内容**：能力、方法、模块、流程、案例都可作为正文主体
- **允许混合表达**：可同时出现概念页、图示页、数据页、案例页
- **收束强调价值**：最后几页回到品牌、结论、建议或下一步动作

### 2.3 与安服胶片的差异

- `chaitin` 不要求每个章节都围绕“证明能力”展开
- 不要求强制采用 `品牌理念 → 服务体系 → 分域能力 → 结果案例 → 背书` 的安服骨架
- 可以根据源文档内容自由决定“先讲方法”“先讲问题”“先讲案例”哪种更合适

---

## 三、页面意图定义

`chaitin` 的正文页重点是 **把复杂内容讲清楚，并保持品牌感**，而不是把每页都做成证据墙。

| 页面意图 | 典型表达任务 | 推荐正文打法 |
|---------|--------------|--------------|
| 概览页 | 快速建立认知框架 | 总结条 + 3-5 个模块卡片 |
| 方法论页 | 解释流程、路径或机制 | 流程图、分层结构、阶段图 |
| 模块展开页 | 说明多个能力点之间的关系 | 栅格卡片、分栏、矩阵 |
| 对比页 | 突出差异、升级、前后变化 | 左右对比、前后对照 |
| 数据页 | 提供关键指标支撑 | KPI + 图表 + 注释 |
| 案例页 | 补强可信度 | 背景 / 动作 / 结果 或 问题 / 方案 / 价值 |
| 收束页 | 再次强调结论或行动方向 | takeaway + 重点摘要 |

## 四、标题逻辑

### 4.1 标题风格

`chaitin` 的标题可以比 `security_service` 更灵活，但仍应避免空泛。

### 4.2 推荐标题模式

| 类型 | 模式 | 示例 |
|------|------|------|
| 主题概览 | `主题 + 结论` | `攻防演练总结：从单点突破走向体系沉淀` |
| 方法论 | `主题：路径/机制` | `攻击队作战机制：目标导向与闭环协同` |
| 模块展开 | `对象 + 结构说明` | `攻击能力体系由四类关键模块构成` |
| 对比页 | `对象A vs 对象B` / `从 X 到 Y` | `从一次性响应到常态化演练机制` |
| 数据页 | `结论 + 指标` | `核心成效：高危问题识别效率显著提升` |

### 4.3 不建议的标题

- `相关介绍`
- `具体说明`
- `工作内容`
- `更多内容`

这些标题过于虚，会削弱通用品牌模板应有的表达力度。

## 五、正文写法

### 5.1 基本原则

- 优先写成 **结构化短句**
- 允许保留少量解释性语句，但不要让一页出现大段连续自然段
- 每个卡片或模块最好都能独立表达一个信息单元
- 同页内容要有明显层级：结论 > 模块标题 > 支撑句

### 5.2 推荐正文形态

适合本模板的正文更像：

```text
作战目标：围绕重点业务与核心资产开展实战验证
关键动作：情报收集、路径设计、自动化验证、人工补强
阶段产出：漏洞清单、攻击路径、风险研判、改进建议
```

而不是：

```text
本次项目围绕多个方向展开了较为全面的工作，
在推进过程中同步结合实际情况进行了不同程度的优化和补充，
最终形成了较为完整的工作成果。
```

### 5.3 与安服模板的差异

- 可以保留适度解释，不必像安服模板那样所有句子都压缩成“证据标签”
- 允许用一页去“讲清楚一个结构”，而不一定非要“证明一个结果”
- 案例和数据是加分项，但不是每一页都必须具备的表达义务

## 六、绘图与布局逻辑

### 6.1 核心绘图套路

| 内容意图 | 推荐图示 | 说明 |
|----------|----------|------|
| 内容概览 | 总结条 + 卡片栅格 | 适合培训、方案、能力介绍 |
| 流程机制 | 时间线 / 阶段流程 / 环形闭环 | 适合方法论和执行流程 |
| 结构说明 | 分层结构图 / 模块关系图 / 矩阵 | 适合体系化表达 |
| 前后对比 | 双栏对照 / 升级路径 | 适合价值说明 |
| 图文结合 | 左图右文 / 上图下文 | 适合案例与场景说明 |
| 数据支撑 | KPI 卡片 + 小图表 | 适合轻量数据页 |

### 6.2 推荐版式节奏

- 同一章节内允许有 2-3 种正文版式交替出现
- 高密度页优先采用 `takeaway + 下层模块` 或 `2x2 / 3x2 卡片`
- 轻量页可以放大单一图示或单一结论
- 如果页面已经出现副标题、总结条或大图，正文块要整体下移，避免上层打架

### 6.3 不推荐的退化形式

- 所有页都做成同一套四卡片，只换文字
- 为了塞内容，把字号一味压小、边距压窄
- 为了追求变化，随意移动 Logo、页码、底部装饰条
- 用过多碎片标签替代真正的结构表达，导致页面看起来“很多字但没组织”

---

## 七、正文布局优先级

由于 `chaitin` 的模板页主要提供固定页眉页脚与品牌骨架，正文页的打法要优先靠内容意图决定。

### 7.1 高优先级打法

- `takeaway + 下层模块`
- 2x2 / 3x2 栅格卡片
- 左右分栏的图文页
- 流程 / 时间线 / 分阶段结构页
- 轻量 KPI + 结论页

### 7.2 中优先级打法

- 单页多卡片列表
- 前后对比页
- 案例三段式页
- 图表 + 结论说明页

### 7.3 低优先级兜底打法

- 纯 bullet list
- 无结构的多段正文
- 单纯堆砌截图而无提炼

---

## 八、生成时的执行提示

当使用 `chaitin` 模板时，Strategist 和 Executor 阶段都应默认执行以下动作：

- 先判断这一页更适合“讲清楚什么”，再决定用哪种模块结构
- 保留品牌骨架稳定，把变化留给正文内容区
- 同一章节内控制版式家族，不要每页都完全重新发明
- 优先用卡片、流程、分栏、对比等结构组织信息，而不是长段文字
- 每页生成后检查：
  - 标题是否具体、有信息量
  - 副标题、takeaway、正文块是否上下打架
  - 文字是否贴边、拥挤、存在视觉溢出风险
  - Logo、页码、底部装饰条是否与正文冲突

## 九、与 `security_service` 的边界总结

一句话区分：

- `chaitin`：**长亭品牌通用底盘，重点是讲清楚、讲丰富、讲有品牌感**
- `security_service`：**长亭安服专项胶片，重点是讲证明、讲结果、讲证据链**

因此：

- 不能把 `chaitin` 当成浅色安服模板的深色版
- 也不能把 `security_service` 当成 `chaitin` 只换一套皮肤
