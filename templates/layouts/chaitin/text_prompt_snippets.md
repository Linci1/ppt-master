# 长亭通用墨绿色文本生成 Prompt 片段

> 用途：给 `chaitin` 模板的 Strategist / Executor 直接拼接使用，约束正文页在保持灵活丰富的同时，不退化成空泛卡片页或安服式证明页。

---

## 一、通用总提示

```text
你正在为长亭通用墨绿色模板生成正文内容。

这个模板的定位是“长亭品牌通用表达底盘”，不是安服专项证明胶片。

请始终遵循以下规则：
1. 每页先明确 页面意图 / 核心信息 / 读者看完应得到的认知
2. 优先解决“这页讲清楚什么”，再决定版式
3. 保留品牌骨架稳定，把变化留给正文内容区
4. 允许丰富表达，但不允许内容松散、标题空泛、结构失控
5. 可以讲方法、模块、对比、流程、案例、建议，但不要强行套成安服证据页

优先使用以下表达方向：
- 主题概览
- 结构说明
- 方法路径
- 模块能力
- 差异对比
- 案例价值
- 行动建议

避免以下问题：
- 所有页都变成同一种四卡片页
- 标题空泛，如“相关介绍”“更多内容”“能力展示”
- 长段大白话说明
- 为了显得专业而堆过多短标签，导致页面碎片化
```

---

## 二、单页提炼提示

```text
请先不要直接写页面正文。

先输出以下 3 项：
1. 页面意图
2. 核心信息（这一页最想讲清楚什么）
3. 页面结构建议（概览 / 流程 / 分层 / 对比 / 案例 / 数据 / 建议）

然后再生成页面正文文本。

要求：
- 标题必须具体、有信息量
- 每个模块都承担一个清晰信息单元
- 同页内容要有层级：标题 > 模块标题 > 支撑句
- 不要先堆内容再临时找结构
```

---

## 三、概览 / 方法页提示

```text
当前页面适合做“讲清楚结构”的正文页。

请优先把内容提炼成：
- 一个总观点
- 3-5 个核心模块
- 每个模块 1 行标题 + 1-2 行说明

推荐表达：
- 这套方法主要由 X、Y、Z 三部分构成
- 页面重点不是罗列功能，而是说明这些模块如何协同
- 读者应一眼看到主线，而不是逐段拼接理解
```

---

## 四、流程 / 机制页提示

```text
请把原始内容提炼成“流程或机制页”。

正文应覆盖：
1. 主流程 / 主机制
2. 阶段划分
3. 每阶段的关键动作
4. 最终输出或价值

要求：
- 时间或流程顺序要清楚
- 图负责顺序，文负责解释和价值
- 不要把流程页写成普通多段说明
```

---

## 五、对比 / 升级页提示

```text
请把原始内容提炼成“对比页”。

正文应清楚说明：
- 对比双方是谁
- 差异点是什么
- 为什么这种差异重要
- 最后推荐哪种方向

要求：
- 不要只列名词对照
- 每个对比点最好带一句解释
- 页面顶部最好有一句总判断
```

---

## 六、案例 / 价值页提示

```text
请把原始内容提炼成“案例或价值说明页”。

正文建议采用：
- 背景 / 动作 / 结果
或
- 问题 / 方案 / 价值

要求：
- 标题不要只是“案例介绍”
- 必须说明这个案例为什么值得讲
- 至少要有一处结果、收益或经验总结
```

---

## 七、高密度页压缩提示

```text
当前页面允许内容丰富，但必须保持品牌节奏和阅读顺序。

请压缩内容时遵循：
- 删掉重复解释
- 把长段拆成结构化短句
- 优先保留主线、模块关系、关键差异、结论
- 若信息已经超载，优先拆模块或拆页，不要无脑缩字

建议控制：
- 模块标题：6-14 字
- 支撑句：1-2 行
- 同页核心模块：3-6 个
```
