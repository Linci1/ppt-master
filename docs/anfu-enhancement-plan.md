# 安服 PPT 生成能力增强方案

> 基于 ppt-master 的安服场景拓展计划
> 
> 更新日期：2026-04-25

---

## 一、总览

### 1.1 目标

在 ppt-master 原有管线基础上，增强安服（安全服务）场景的正文页生成质量，使其产出接近真实安服 PPT 的视觉水准。

### 1.2 核心思路

**正文页靠 LLM 完成，核心杠杆是"套路性提示词"**——把安服案例中反复出现的排版模式提炼成可复用的布局提示词模板，给 Executor 参考。

不改路径、不改 SVG 模板结构、不硬编码布局。通过提示词让 LLM 在 `03_content.svg` 的大框内做出不同布局。

### 1.3 当前管线

```
源材料 → Step1:材料转MD → Step2:项目初始化 → Step3:模板选择+图片提取
→ Step4:Strategist(八项确认+design_spec) → Step5:Image_Generator(可选)
→ Step6:Executor(SVG逐页生成) → Step7:后处理+导出PPTX
```

正文页生成路径：LLM 读取 design_spec + layout-patterns-security.md → 逐页写 SVG → finalize → svg_to_pptx.py 导出。

### 1.4 风格基准（2案例蒸馏）

| 维度 | 蒸馏结论 |
|------|----------|
| 背景 | **正文页白底** #FFFFFF，暗色仅限封面/章节/结尾页 |
| 主色 | 两套：品牌绿 #7BBD4A（安服胶片·默认）/ 青绿 #43827F（HW总结） |
| 告警 | 红 #FF0000 / #C00000，黄 #FFFF00 |
| 正文 | #404040，字号 12pt(16px) |
| 图片 | **95%正文页含图**，img_right 占 50%+，91%图片面积<10%（小截图为主） |
| 标题 | 92%在顶部窄条（y=0~1.5"） |
| 章节页 | 极简（大号数字+标题，2-3个元素） |

详细蒸馏报告：`references/chaitin-style-distillation.md`

1. **排版范式库**：安服场景的5种高频排版范式
   - 攻击链复盘页（水平时间轴 + 详情卡片）
   - 漏洞矩阵页（网格卡片 + 严重度色标）
   - 红蓝对比页（双栏 + 对立色系）
   - 资产风险页（分类卡片 + 风险等级标记）

3. **讲稿风格**：技术叙事 + 业务影响翻译

4. **图片策略**：强制嵌入产品截图 / SVG 自绘示意图
   - 每页至少1张图
   - 优先 img_right
   - 无素材时用 SVG 绘制流程图/架构图

### 方案3：套路性提示词（最核心杠杆）

**目标**：把安服案例中反复出现的排版模式提炼成可复用的"布局提示词模板"，给 Executor 参考。

这是解决"03_content.svg 只有一个大框"问题的正确方式——不硬改模板，通过提示词让 LLM 在大框内做出不同布局。

#### 3.1 套路提示词的定义

套路提示词是一段高层排版指令，描述某类页面的布局结构、组件类型、颜色规则、适用场景。格式如下：

```
布局模式：攻击链复盘页
结构：顶部标题 → 水平时间轴(6节点) → 底部2-3个详情卡片
时间轴：每个节点=圆形徽章(品牌色填充)+事件名(12px)，节点间用连接线
详情卡片：对应关键节点的技术细节，卡片内=标题(16px bold)+3-4行正文(14px)
颜色：主节点用品牌绿，辅助节点用灰，高危节点用红
适用：渗透测试复盘、攻防演练总结、应急响应过程
```

这种提示词模板可以做到：
- **稳定性**：每次遇到同类内容，LLM 拿到相同的布局指引，输出结构一致
- **丰富性**：不同页匹配不同模板，不会千篇一律
- **风格化**：模板内嵌了颜色/字号/间距规范，保证品牌一致

#### 3.2 工作流

1. 从安服 PPT 中提取高频页面类型（已完成：2案例152页分析）
2. 每种类型提炼一个布局提示词模板（**进行中**，当前完成度 40%）
3. Strategist 阶段根据内容自动匹配模板，写入 design_spec（**未开始**）
4. Executor 阶段严格按模板生成（**基础框架已有**，需细化）

#### 3.3 当前 layout-patterns-security.md 的内容层级

当前文件包含两个层级的内容：

| 层级 | 内容 | 完成度 | 说明 |
|------|------|--------|------|
| **L1 框架层** | 10种布局类型 + 决策树 + 图片强制规则 + 关键纪律 | **85%** | 已完成白底修正、img_right优先、custom出口 |
| **L2 坐标层** | 每种布局的坐标规范（文字栏x/y/w、图片栏x/y/w） | **90%** | lr_split_imagetext 已有完整坐标 |
| **L3 套路层** | 高层排版指令模板（"攻击链复盘页"式描述） | **40%** | **核心缺失**：只有坐标，无高层套路描述 |

L3 层是效果最大的单点，也是方案3的核心交付物。

## 七、风险与约束

1. **LLM 排版想象力有限**：即使有套路提示词，LLM 对空间布局的理解仍不如人类。需要后置检查兜底。
2. **套路模板 vs 灵活性的平衡**：模板太多会限制创造力，太少则千篇一律。当前 10 种布局 + 8 种套路，需要验证是否足够。
3. **executor-security 维护成本**：新增角色需持续维护排版范式库。建议先草案试跑，不急于固化。
4. **图片素材质量**：提取的源文档图片可能有低分辨率、截图工具 UI 痕迹等问题。需要在 Step 3 增加质量筛选。
5. **SVG→PPTX 信息丢失**：正文页字号丢失（0.1pt）问题仍存在。当前通过 finalize_svg.py 部分修复，但未根治。长尾方案可能需要路径B（python-pptx 原生）混合使用。
